## Getting Started

<hr />

### Generate a new fullstack Rapid project with nextjs or remix:
<div className='mt-2'></div>
> This will give you a project scaffold containing all the needed files to develop an application with either Nextjs or Remix (use "rapid new --remix" to scaffold with Remix instead of Nextjs)
<CodeBlock language='sh' code={`rapid new --nextjs`} />

<br />
<br />
## Start your application:
<hr />

### Run the backend:

<CodeBlock language='sh' code={`rapid run `} />

<br />
### Run the frontend:
<CodeBlock language='sh' code={`npm run dev`} />


<br />
<br />
## File structure
<hr />
The Rapid scaffolds have an almost identical structure to the defaults that you get with both the `nextjs` and `remix` CLIs. The only thing that is different is the `/api` directory.
